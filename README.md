# DeepSeek-AutoGen 多智能体协作系统 使用说明
本项目基于 **AutoGen 多智能体框架** 与 **DeepSeek 大模型** 开发，包含**网页可视化版**和**命令行极简版**两套独立运行的程序，均通过「需求分析→内容生成→审核优化」三个智能体自动完成编程、文案类任务，所有生成内容自动本地保存，可直接用于学习、演示与课程作业。

## 一、项目完整结构
```
AUTOGEN_DEMO/
├── .cache/                # AutoGen框架运行缓存，自动生成，无需修改
│   └── 41/
│       └── cache.db       # 框架缓存数据文件
├── agent_output/          # 两个版本共用的输出根目录
│   ├── codes/             # 存放所有生成的代码文件（.py/.cpp/.java等）
│   ├── docs/              # 存放所有生成的文案、Markdown文档
│   ├── logs/              # 仅命令行版使用：任务执行日志（JSON格式）
│   ├── reports/           # 仅命令行版使用：自动生成的任务报告（Markdown）
│   └── zip/               # 网页版预留：文件压缩包存储目录
├── task_sessions/         # 网页版专用：按任务ID存储会话记录、聊天历史
├── templates/             # Flask固定模板目录（必须保留此名称）
│   └── index.html         # 网页版前端页面，与APP.py配套使用
├── user_sessions/         # 网页版专用：按用户UUID存储会话数据，多用户不冲突
├── .env                   # 配置文件（需手动创建），存储DeepSeek API密钥
├── .gitignore             # Git忽略配置，屏蔽缓存、日志等无需上传的文件
├── APP.py                 # 网页版核心后端程序（Flask+多智能体+接口）
├── chat_history.json      # 网页版全局聊天记录，自动生成
├── deepseek_autogen_demo.py  # 命令行版独立程序，无前端依赖
└── task_counter.json      # 网页版任务编号计数器，自动维护，请勿删除
```

## 二、环境配置
### 1. 安装依赖包
打开命令提示符/终端，进入项目根目录，依次执行以下命令：
```bash
# 安装两个版本通用依赖
pip install pyautogen python-dotenv
# 网页版额外需要Flask框架
pip install flask
```

### 2. 配置API密钥
1. 在项目**根目录**新建文本文档，重命名为 `.env`
2. 打开 `.env` 文件，填入以下内容（替换为你的DeepSeek API密钥）：
```env
DEEPSEEK_API_KEY=你的DeepSeek官方API密钥
```
3. 保存关闭，配置完成。

## 三、网页可视化版（APP.py + templates/index.html）
### 1. 核心功能
#### 后端功能（APP.py）
- 搭建Flask网页服务，对接前端所有操作请求
- 三个智能体分工协作：分析需求、生成内容、优化审核
- 支持多用户同时使用，通过UUID隔离会话，数据不混淆
- 自动保存任务记录、聊天内容，任务编号连续不重复
- 自动生成代码/文档文件，支持预览、下载
- 内置Python代码语法校验，提示代码错误位置

#### 前端功能（index.html）
- 可视化对话界面，支持Markdown渲染、代码语法高亮
- 侧边栏实时显示：任务ID、Token消耗、预估费用、执行耗时
- 历史任务管理：加载历史对话、删除任务、折叠任务列表
- 亮色/暗色主题一键切换，设置自动保存
- 快捷键操作：Enter发送消息，Shift+Enter换行输入
- 加载动画、提示弹窗，操作更直观

### 2. 运行步骤
1. 确认 `index.html` 放在 `templates` 文件夹内
2. 打开终端，进入项目根目录，执行启动命令：
```bash
python APP.py
```
3. 启动成功后，打开浏览器，访问地址：
```
http://127.0.0.1:5000
```

### 3. 操作说明
- 发送需求：在输入框填写编程/文案需求，按Enter发送
- 新建会话：点击「新建会话」，自动保存上一轮对话并清空当前页面
- 历史任务：侧边栏查看所有已保存任务，点击可加载历史对话
- 文件操作：生成的代码/文档可直接预览、下载到本地
- 主题切换：点击「切换主题」，在亮色/暗色模式间切换

### 4. 自动生成文件规则
- 代码文件：`task_任务ID_文件名.后缀`、`task_任务ID_文件名_opt.后缀`
- 文档文件：`task_任务ID_文件名_正文.md`、`task_任务ID_文件名_优化版.md`
- 会话文件：存储在 `task_sessions`、`user_sessions` 目录，JSON格式

## 四、命令行极简版（deepseek_autogen_demo.py）
### 1. 核心功能
- 无网页界面，纯控制台运行，轻量化无依赖
- 三个智能体自动完成需求分析、内容生成、优化审核
- API请求异常自动重试，避免网络问题导致运行失败
- 控制台彩色输出，区分代码、文案、提示信息
- 仅在需求包含「保存/导出/生成文件」时，自动保存文件
- 自动生成任务日志、Markdown任务报告
- 实时统计Token、费用，退出程序时显示总消耗

### 2. 运行步骤
1. 确保 `.env` 配置文件已填写正确API密钥
2. 打开终端，进入项目根目录，执行命令：
```bash
python deepseek_autogen_demo.py
```

### 3. 控制台指令说明
- 直接输入文字：提交任务需求
- 输入 `new`：清空当前对话上下文，重新开始
- 输入 `exit`/`quit`/`q`：退出程序，显示本次运行总消耗
- 输入 `help`：查看使用帮助

### 4. 输出文件说明
- 代码/文档：保存在 `agent_output/codes`、`agent_output/docs`
- 任务日志：保存在 `agent_output/logs`，JSON格式记录任务详情
- 任务报告：保存在 `agent_output/reports`，Markdown格式可直接查看

## 五、智能体分工说明
### 1. 需求分析智能体
- 判断任务类型：编程任务 / 文案任务
- 识别指定语言（如C++/Java/Python）
- 精简提炼核心需求，不生成实际内容

### 2. 内容生成智能体
- 按需求生成代码/文案，严格遵循指定语言
- 代码用标准格式包裹，无多余解释文字
- 文案结构完整、语言通顺

### 3. 审核优化智能体
- 代码优化、生成测试用例，保持原语言不变
- 文案润色、理顺逻辑、修正语句
- 仅输出优化后内容，无额外说明

## 六、关键注意事项
1. **文件路径不可修改**
`index.html` 必须放在 `templates` 文件夹内，否则APP.py无法加载网页。
2. **禁止手动修改自动生成文件**
`agent_output`、`task_sessions`、`user_sessions` 内的文件均由程序自动管理，手动修改会导致数据混乱。
3. **两个版本完全独立**
网页版和命令行版可单独运行、同时运行，互不干扰，共用输出目录。
4. **任务编号规则**
网页版任务编号由 `task_counter.json` 自动维护，重启程序会延续之前编号，请勿删除此文件。
5. **文件保存条件**
命令行版需在需求中加入「保存/导出」，才会生成本地文件；网页版新建会话时自动保存。

## 七、常见问题与解决方法
1. **网页打不开，提示找不到页面**
检查 `index.html` 是否在 `templates` 文件夹内，确认APP.py已正常启动。
2. **运行报错，提示API密钥错误**
打开 `.env` 文件，检查 `DEEPSEEK_API_KEY` 是否填写正确，无多余空格。
3. **找不到生成的文件**
代码文件在 `agent_output/codes`，文档文件在 `agent_output/docs`。
4. **代码提示语法错误**
仅Python代码会自动校验，其他语言不校验，属于正常情况。
5. **任务编号不连续**
由程序自动计数器维护，属于正常现象，不影响使用。
